# Agent Desktop Constructor

`agent_desktop_constructor` - Python desktop-приложение для конструктора ИИ-агентов.

Пользователь описывает агента текстом, а приложение должно строить проверяемую спецификацию `AgentSpec`. Исполнение выполняется через `Agent Runtime`, который обрабатывает спецификацию как граф шагов. На этом этапе создана только чистая структура проекта без бизнес-логики, COM-интеграций и LLM.

## Технологии

- Python 3.11+
- PySide6 для desktop UI
- Pydantic для схем данных
- SQLAlchemy для локального хранения
- pytest для тестов
- pywin32 как условная зависимость только для Windows

## Архитектура

Проект закладывает следующие слои:

- `app`: точка входа приложения.
- `core`: общие модели, сервисы и ошибки.
- `builder`: будущая сборка и валидация `AgentSpec`.
- `runtime`: будущий исполнитель графа агента.
- `tools`: будущий Tool Gateway и Tool Registry.
- `workers`: будущие изолированные worker-процессы для внешних интеграций.
- `storage`: будущий слой локального хранения состояния и журналов.
- `ui`: будущий PySide6-интерфейс без бизнес-логики.
- `tests`: тесты проекта.

## Каталог инструментов

Список доступных инструментов хранится в
`agent_desktop_constructor/tools/default_tools_catalog.json`. Каталог является
единым источником правды для UI, `AgentBuilder`, будущего Planner и проверки
готового `AgentSpec`.

`AgentBuilder` и LLM-планировщик должны выбирать только инструменты из каталога.
Запрещено придумывать новые `tool_name`: если подходящего инструмента нет,
агент должен запросить человека или вернуть потребность в новом инструменте.
Свойства риска (`read`, `create_draft`, `write`, `dangerous`) и необходимость
подтверждения человека также берутся из каталога, а не дублируются вручную.

## Запуск

```bash
python -m agent_desktop_constructor.app.main
```

Пока приложение только выводит сообщение о старте.
